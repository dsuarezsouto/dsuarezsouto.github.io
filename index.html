dreamPy

Twitter, Web Scrapping, NLP, Elastic, Statistical Modelling

Twitter is a social network that allows its users to exchange messages of 280 characters with the possibility of accompanying them with a photo, video and/or link. This social network has been used as a source of data for numerous research studies on the human being. This study aims to analyse and characterize the messages coming from Spanish-speaking users related to the most common sleep disorder in our society, insomnia. For this purpose, this study provides two machine learning classifiers that enable the classification of users with insomnia together with the self-reported cause. For training these classifiers, a dataset of tweets in Spanish containing the word insomnia has been manually annotated to draw conclusions about the geographical distribution, symptoms and the different topics that users with insomnia treat. In addition, a second dataset has been collected formed by two groups of users from Spain with insomnia and without insomnia. Analysing the timeline of both groups we have been able to extract the differences in the patterns of activity on Twitter of each of these groups. For its automation, a data pipeline has been developed for the capture, analysis and visualization of the tweets related to insomnia.

The Taxi Oracle

EDA, Data Preparation, Business Analysis

The main goal of this project is to create a model that will predict the demand of taxis per hour and location that a private transportation company will need to perform trips, at a given location and time. The performance of this model will be analyzed not just in terms of prediction error (MAE and MSE), but also in terms of financial profit. These predictions will allow taxi companies minimizing the waiting time of customers. Other external factors that have to be considered to design the best model include weather and holidays. Extreme weather conditions affecting the customers’ rapid service. Public holidays like Thanksgiving, Christmas etc. and weekends where the demand of taxis would be higher than usual. House Price Analysis

Regression Analysis, EDA, Statistical Modelling

The objective of this project is to perform an analysis and a prediction model for the price of homes in the city of Washington. Some of the findings of this project have been the different impacts of house characteristics on house prices. We found that the year of sale is the most important factor in predicting price, followed by the year the construction is completed. In addition, with respect to the number and type of rooms, the biggest increase in price is found when going from 1 bathroom to 2 (with an average increase of 33%). Regarding the prediction model, the best results have been obtained with Random Forest (with an R squared of 0.82 in the test set). A detailed report of the analysis can be found here.

Plant Pathology - FGVC7

Computer Vision, CNN, Tensorflow, Fine-Grained

In this project we will implement the approach of the ELoPE paper in order to obtain the best possible position in the Pathology 2020 - FGVC7 Kaggle competition. In this Kaggle competition we are given images of plants, which contains 1,821 1800 x 2100 color images from 4 different classes: healthy, scab, rust or multiple diseases. The test set contains another 1,821 images, and the metric used to evaluate this test set is going to be the Mean Columnwise Area Under Receiver Operating Caracteristic Curve. Our solution is based on a CNN backbone, with Global K-Max Pooling and an Embedding layer for the calculation of the loss with respect to two terms, within class loss and between class loss. A detailed report of the analysis can be found here.

PROY

Web Server, Java

PROY is a web application that is dedicated to manage the information of hours that the personnel of a company dedicates to the projects or activities that it carries out. It consists on a web application that will be used by every person related to the project, from the developers who are part of the development team to the HR department. In addition, it is proposed to have distributed access via web, Smartphone, etc. The system tries to make it easier for companies to manage engineering projects in a fast and efficient using an intuitive interface that is different depending on the user who is using the application. In addition, the application may be connected to Google Calendar. You can also generate reports directly from the web and export them as pdf.
